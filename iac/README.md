# Лабораторна робота №4 — IaC (Terraform + Ansible)

Студент: Бігіч Назар, варіант **N = 5** (V2=2 → TOML + PostgreSQL, V3=3 → Simple Inventory, V5=1 → port 8080).

## Архітектура

```
                  +-------- VM1 worker (lab4-worker) -------+    +--- VM2 db (lab4-db) ---+
client → :80 →    | nginx 0.0.0.0:80  →  app 127.0.0.1:8080 | →  | PostgreSQL 5432       |
                  +-----------------------------------------+    +-----------------------+
                              ↑                                     ↑
                              └──────── private 10.0.1.0/24 ────────┘
```

Реалізовано в **2 етапи**:

1. **Terraform** (`terraform/`) — провіжен 2 Hetzner Cloud VM (Ubuntu 24.04, CX22, в одній приватній мережі `10.0.1.0/24`), cloud-init інжектить SSH-ключ + користувача `ansible` з NOPASSWD sudo.
2. **Ansible** (`ansible/`) — конфігурація ВМ через 4 ролі: `common`, `db`, `worker`, `verify`. Jinja2 шаблони для `config.toml`, nginx, sudoers, compose, systemd.

## Передумови

- Terraform ≥ 1.6
- Ansible ≥ 2.16 + колекції `community.postgresql`, `ansible.posix`
- Hetzner Cloud API token (Read & Write) — отримати в [console.hetzner.cloud](https://console.hetzner.cloud) → Security → API Tokens
- SSH ключ (наприклад `~/.ssh/id_ed25519.pub`)

## Запуск

### 1. Provisioning (Terraform)

```bash
cd iac/terraform
cp example.tfvars terraform.tfvars
# відредагуй terraform.tfvars: hcloud_token + ssh_public_key

terraform init
terraform fmt -check
terraform validate
terraform plan
terraform apply
```

Виведе `worker_public_ip` + `db_public_ip` + згенерує `../ansible/inventory/hosts.ini`.

### 2. Vault (один раз, до першого playbook run)

```bash
cd iac/ansible
mkdir -p inventory/group_vars

openssl rand -hex 24 > /tmp/dbp
ansible-vault encrypt_string "$(cat /tmp/dbp)" --name vault_db_password \
  > inventory/group_vars/vault.yml
shred -u /tmp/dbp

# додай vault password file
echo 'MY-VAULT-PASSWORD' > .vault_pass
chmod 600 .vault_pass
```

### 3. Configuration (Ansible)

```bash
cd iac/ansible
ansible-galaxy install -r requirements.yml
ansible-playbook -i inventory/hosts.ini site.yml --vault-password-file .vault_pass
```

Перший прогін піднімає все. Другий прогін має бути `changed=0` (ідемпотентність).

### 4. Перевірка

```bash
WORKER=$(cd ../terraform && terraform output -raw worker_public_ip)
DB=$(cd ../terraform && terraform output -raw db_public_ip)

curl -sS http://$WORKER/items
curl -sS -X POST -H 'Content-Type: application/json' -d '{"name":"bolt","quantity":3}' http://$WORKER/items
curl -sS http://$WORKER/items/1

curl -sS -o /dev/null -w "%{http_code}\n" http://$WORKER/admin           # 404
curl -sS -o /dev/null -w "%{http_code}\n" http://$WORKER/health/alive    # 404 (ззовні блокується)

# DB зовні недоступна:
nc -zv $DB 5432   # connection refused or timeout

# Юзери:
ssh ansible@$WORKER  whoami      # ansible (NOPASSWD sudo)
ssh teacher@$WORKER  whoami      # запитає пароль 12345678
ssh operator@$WORKER whoami      # пароль 12345678
ssh operator@$WORKER sudo -n -l  # тільки 6 команд із MYWEBAPP_CTL alias

cat /home/student/gradebook      # 5
```

### 5. Cleanup

```bash
cd iac/terraform
terraform destroy
```

## Структура

```
iac/
├── terraform/
│   ├── versions.tf                       # required providers (hcloud, local)
│   ├── variables.tf                      # tunable knobs (token, key, locations, IPs)
│   ├── main.tf                           # network/subnet/firewalls/servers/inventory render
│   ├── outputs.tf                        # IPs + inventory path
│   ├── cloud-init.ansible.yaml.tftpl     # creates ansible user via cloud-init
│   ├── inventory.ini.tftpl               # generated Ansible inventory
│   ├── example.tfvars                    # template with safe placeholders
│   └── .gitignore                        # excludes state + real *.tfvars
└── ansible/
    ├── ansible.cfg
    ├── requirements.yml                  # collections
    ├── site.yml                          # top-level orchestration
    ├── inventory/
    │   ├── hosts.ini                     # ← generated by Terraform (gitignored)
    │   └── group_vars/
    │       ├── all.yml                   # image SSOT, gradebook, default passwords
    │       └── vault.yml.example         # vault template (real one is gitignored)
    └── roles/
        ├── common/        (teacher, gradebook, apt hardening)
        ├── db/            (postgresql + role + pg_hba)
        ├── worker/        (docker + compose + nginx + users + sudoers)
        └── verify/        (post-deploy uri assertions)
```

## Спецвимоги — мапа

| Критерій спеки | Артефакт | Перевірка |
|---|---|---|
| 1. `terraform apply` піднімає 2 ВМ | `terraform/main.tf` `hcloud_server.worker` + `.db` | `terraform plan` показує 2 servers |
| 2. `ansible-playbook` конфігурує | `ansible/site.yml` 5 плеїв | Прогін без помилок |
| 3. Ідемпотентність | Усі задачі через stdlib модулі (apt, user, template, systemd, postgresql_*) | Другий прогін: `changed=0 failed=0` |
| 4. Декларативність — без `shell`/`command` | Лише одна `ansible.builtin.shell` (детект версії pg, `changed_when: false`) | grep підтверджує |
| 5. DB лише з worker | `pg_hba.conf.j2`: `host mywebapp mywebapp 10.0.1.10/32 scram-sha-256` + `reject` для решти | `nc -zv $DB 5432` ззовні → refused |
| 6. SSH для ansible/teacher/operator + restricted sudo | `common` (teacher), `worker/users.yml` + `worker/sudoers.yml` | `sudo -n -l -U operator` показує лише `MYWEBAPP_CTL` |
| 7. `/health/alive` 200, `/health/ready` 200 (worker→db) | `app/routes/health.py` + `config.toml.j2` з приватним IP db | `curl 127.0.0.1:8080/health/ready` на worker → 200 |
